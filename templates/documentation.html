RESTful API: Upscale Images & Extract Text/Data Documentation

August 6,2025 by Tristan Do

This API provides endpoints to perform three main tasks:

  1. Image Upscaling — Improve the resolution and clarity of input images using advanced AI models.

  2. Image-to-Text Extraction — Extract structured or unstructured text from various image types, including:

    • Handwritten Notes
    • Printed Documents

  3. Image-to-Table Extraction — Extract structured data from various image types as tables and spreadsheets.

Results can be exported as .txt or .csv files, depending on the content and your requested output.

Use Cases

  • Restore and upscale low-res images (e.g., for achival or printing)
  • Digitize handwritten class notes or meeting summaries
  • Extract data from paper invoices, receipts, or form snapshots
  • Conver scanned tables into structured spreadsheets

Output Formats

  • Text Documents (.txt)
  • Comma-Separated Values (.csv)
  • Optional raw JSON (for developers)

Authorization: Bearer Token


Endpoints & Usage

GET — Get Image History

http://127.0.0.1:5000/v1/history_images

Make a GET request to retrieve all existing image files that have been upscaled from this API in the current database.

Authorization: Bearer Token {vault:auth_token}

GET — Get Text History

http://127.0.0.1:5000/v1/history_text

Retrieve all existing text files transcribed from this API in the current database.

Authorization: Bearer Token

GET — Get History Tables

http://127.0.0.1:5000/v1/history_tables

Retrieve all existing datatable files recorded from this API.

Authorization: Bearer Token

GET — Find Extract File

http://127.0.0.1:5000/v1/find_file?id={file_id}

Find a specific file in the history by file ID.

Query Params:

  • id: int — The file ID

Authorization: Bearer Token

GET — Query Image

http://127.0.0.1:5000/v1/query_image?keyword={keyword}&date={date}

Search the image database for metadata and file info.

Query Params:

  • keyword: str
  • date: str

Authorization: Bearer Token

GET — Query Table

http://127.0.0.1:5000/v1/query_table?keyword={keyword}&date={date}

Search for datatable files and get related metadata.

Query Params:

  • keyword: str
  • date: str

Authorization: Bearer Token

GET — Query Text

http://127.0.0.1:5000/v1/query_text?keyword={keyword}&date={date}

Query the text file database for historical logs and file metadata.

Query Params:

  • keyword: str
  • date: str

Authorization: Bearer Token

POST — Add Image

http://127.0.0.1:5000/v1/add_image

Upscale image(s) and store pre/post versions. Max 10 images recommended for speed. Files must be under 100KB each for optimal performance.

Body: form-data

  • images[]: image.png, image.jpg, image.jpeg, etc.

Authorization: Bearer Token

POST — Add Table

http://127.0.0.1:5000/v1/add_table

Extract table data from images and return as CSV or JSON (or zip multiple files).

Body: form-data

  • dataextract: image.jpg, image.png, etc.

Authorization: Bearer Token

POST — Add Text

http://127.0.0.1:5000/v1/add_text

Extract text from handwritten or printed images. Optionally AI-optimize for accuracy.

Body: form-data

  • textract: image.jpg, image.png, image.jpeg, etc.

Authorization: Bearer Token

PATCH — Fix Text File

http://127.0.0.1:5000/v1/fix/text

Edit an existing text file’s transcription if incorrect. Override previous AI-generated result.

Body: raw (JSON)

  • id: int - file id
  • text: str - text replace content

Authorization: Bearer Token

PUT — Replace Data Table

http://127.0.0.1:5000/v1/replace/datatable

Upload a new version of a previously extracted table. Replaces content using same file ID.

Body: form-data

  • file: .csv, .json file
  • id: int - file id

Authorization: Bearer Token

DELETE — Delete Extract File

http://127.0.0.1:5000/v1/delete/<int:file_id>

Delete a specific file using its ID.

Authorization: Basic Auth
Username: <username>
Password: {vault:user_pass}

DELETE — Clear Database

http://127.0.0.1:5000/v1/clear

⚠ TWO-STEP AUTHENTICATION REQUIRED.
Completely wipe all files and history: upscaled images, extracted text, and tables.

Authorization: Basic Auth
Username: <username>
Password: {vault:user_pass}
